# Overview

## What This Is

`magic_storage` — NeoForge 1.21.1 的一站式儲存 + 合成 mod。玩家用一個終端完成儲存、取用與合成:無限容量(只限物品種類數)、配方書式一鍵合成、以及由 Fuel 頁已安裝機器產生、合成時一次消費的「合成能量」。

概念源自 **Terraria 的 Magic Storage**(無限容量、配方書操作);NeoForge 的儲存網路 / grid UI / 物品處理等**技術模式參考 Refined Storage 2** 原始碼(見 `docs/notes.md`)。

## Key Concepts / Domain

- **Storage Core** — 網路中樞,所有計算(容量、合成、能量)都在 Core;Unit 只提供空間。
- **Storage Unit** — 六個 Tier 的儲存方塊,獨立、不可逆、可並聯不限數量。
- **Crafting Energy / Stations & Axe Energy** — 多個能量池統一用 `Map<EnergyType, Long>`。`MachineEnergyTable` 將九個 Fuel-page descriptors 分成三類：Furnace/Blast Furnace/Smoker/Campfire/Brewing Stand 是可堆疊的 process machines，stack `N` 為對應 pool 產生 `N/tick`；Crafting Table/Stonecutter/Smithing Table 是最多各一個、可取回且只解鎖配方的 instant stations；axe 是不可安裝/取回的 consumable input。Core 持久化前八個 machine/station stacks、finite `axeEnergy` 與 explicit `infiniteAxeEnergy`；有限值為 remaining durability × (vanilla Unbreaking level + 1)，Mending/其他附魔不加值，checked overflow 不吃物品，Unbreakable 轉為永不遞減的無限值。舊 slot 8 axe 只在轉換成功後清除；失敗時同一 Axe Energy 格會代理原物供取回。舊版堆疊 instant station 會保留一個安裝，extras 搬回 Core 一般儲存。`ItemStack#getBurnTime(null)` 是 Fuel 真值；具體 cooking recipe 的 `cookingtime` 同時決定 process/Fuel 消耗。內部 `furnace_fuel` ID 不變，玩家只看到 **Fuel**。所有 mutation 在 server，client 只接同步。
- **Crafting/Fuel GUI** — `TerminalProfile` 以 capability 組合 Storage reduced profile 與 Crafting full profile，`TerminalLayout.forProfile(...)` 是唯一 geometry entry point；同一個 immutable `TerminalLayout.Geometry` 驅動 rendering、widgets、slot reconstruction、hitboxes、tooltips、scroll 與 EMI exclusions。`StorageTerminalScreen` 是共用 shell，擁有 item grid/search/scroll、左 rail 背景、Name/Quantity/Mod/ID controls、focus cleanup 與 amount overlay；`CraftingTerminalScreen` 只提供 pages、player-inventory source、output destination、recipe/Fuel controls，不再先建立父類 widgets 再隱藏重做。所有 rail buttons 固定 18×18、icon canvas 固定 16×16；page/source/output 優先用語意 item，抽象排序/搜尋/箭頭來自單一 `terminal_controls.png` atlas。共用 cycle 契約是左鍵下一個、右鍵上一個、wheel down 下一個、wheel up 上一個，tooltip 只顯示目前值與操作提示；Fuel target 也使用相同 input contract，但 server 仍只接受 Auto/exact target IDs。Fuel target旁另有由同一 ordered descriptors 產生的 `PopupList`：顯示代表物、localized name與 selected state，最多六列後 bounded scroll，選取/outside/Escape/離頁會關閉並清 focus；開啟時加入 EMI exclusion且攔截底層 slot click/tooltip。Machine/reserve tooltip只在實際 18px slot/16px代表 icon 上觸發。Storage/Craftable synthetic display stack 攜帶 server-owned exact `long` amount metadata，Craftable 顯示目前 Core 庫存量，零庫存仍可見但不畫數字；兩頁共用 deterministic comparator。所有 grid amount 先 compact-format，再以畫面初始化時計算的一個固定 scale 在自己的 16px bound 內右下對齊，不再逐值切換正常/半尺寸，exact long 留在 tooltip。Recipe workspace 只消費 server 建立並同步的 immutable `RecipePresentation`：exact recipe ID/kind、shaped dimensions/positions、shapeless state、精確 output components/count、station，以及 item/energy/tool available-required rows；tool row 另同步 explicit infinity bit，所以有限 `Long.MAX_VALUE` 與無限 reserve 不混用。Client 不掃 `RecipeManager` 或 Core。版面垂直分成 diagram、可變長 ledger、footer；exact standard recipe 在 required client-side EMI 中由 guarded compat adapter 透過公開 `EmiRecipe#addWidgets`/`Widget`/`WidgetHolder` 渲染並只在 diagram bounds 轉送 input/tooltip，無 EMI 或 internal synthetic axe 才選 native renderer，unexpected EMI exception 不會靜默 fallback。Ledger 與 craft controls 始終由 Magic Storage 擁有，item rows 用中性底色，energy/tool rows 用深紅底色。Fuel 的 **Stations & Axe Energy** 維持兩個 flow rows；process/instant descriptors 綁 Core station slots，consumable axe descriptor 通常綁只轉換、不保存物品的 transient input，legacy conversion 失敗時則先顯示可取回原物，並顯示 server-synced finite/∞ reserve。Wide machine/reserve panels 使用完整 inner width，右下 `fuelControlPanel` 與 player inventory 同高；descriptor overflow 才採 panel-local paging。數量 actions 為 live-disabled `×1/×8/×64` 加 server-revalidated `Max`。目前 descriptors 仍由程式碼定義，跨 mod runtime registration/sync API 列入 roadmap。
- **Terminal(三階)** — `storage_terminal`(T1 取存)、`crafting_terminal`(T2 配方+一鍵合成)、`remote_terminal`(T3 物品,同維度遠端存取；綁 exact Core UUID,同座標 replacement 不會被靜默接管；重綁不覆蓋其他 `CUSTOM_DATA`)。
- **Network** — Core 掃描/維護 Core ↔ Unit ↔ Terminal/Bus 連通;一般/指令/活塞放置會合併到 next-tick topology pass，安全範圍內增量成長，破壞/不確定拓樸時 full rebuild。Terminal/Bus 只沿目前已載入且仍為 network block 的 cached path 工作，失效時改找另一條已載入路徑，絕不為驗證連線載入 chunk；Import/Export Bus 做自動化。
- **Supported recipes / safe output** — Vanilla exact-concrete Shaped/Shapeless Crafting、Stonecutting、Smelting、Blasting、Smoking、Campfire、component-exact `SmithingTransformRecipe`，以及只掃 `minecraft` namespace default state 的 internal synthetic axe strip/scrape/wax-off；每種都先要求目前 Core 安裝 matching station，或具備足夠 Axe Energy。Smithing 三個 role 直接用原 recipe predicate 比對實際完整 stack，不用 registry-default Ingredient 近似重建。Preview/execute 從目前 server `RecipeManager` 或依 Core Axe Energy 開啟的 exact synthetic axe catalog 重取 holder。Craftable 由 server recipe catalog 產生目前可合成的 output union，display amount 只取該 output 的目前 Core 庫存；synthetic marker 在建立 `ItemKey` 或實際輸出前會移除。Direct `×1/×8/×64/Max` 使用獨立、server-synced 的 Player/Storage session destination：Player 先放玩家 36 格、overflow 回 Core；Storage 則讓 primary output 與 crafting remainder 全部只進 Core。所選策略無法接收完整批次時在 mutation 前整批拒絕，不掉落、不消失。EMI 傳 exact recipe ID + amount + destination，NONE 只選取，CURSOR/INVENTORY 執行單層 server-authoritative magic craft，且不受 direct-terminal toggle 影響。Max 先計算資源上限，再找出 destination 可完整接收的最大 craft 數；midpoint 與每個 Core key 的 reservation/extraction/output/rollback 全程使用 long-count mutation，連 `Long.MAX_VALUE` 都不會退化成逐 `Integer.MAX_VALUE` chunk loop。Commit 在 deferred-listener mutation batch 內驗證、扣料、插入 output、扣 process/Fuel/Axe Energy，再套用玩家 inventory/cursor；finite Axe Energy 每 craft 精確扣一，infinite 不遞減。仍不做 RS2 recursive pattern tree；Smithing Trim、special/dynamic subclasses、Brewing、任意 mod block hook 與自由 world/player-context recipe 目前不執行。
- **Assets** — 所有 block/item model 實際引用的 mod texture 都是 vanilla resolution 16×16 RGBA，並共用固定 dark-stone 機殼像素與 cyan/amethyst palette。Core、Storage/Crafting terminals、T1–T6 容量條、Import/Export 方向與 Remote 各有固定語意；`terminal_controls.png` 以 16px cells 統一 A/bar/grid/#/magnifier/@/arrow glyph。`art/texture-generation/20260714-terminal-family/` 保存 Replicate predictions、固定 seed/img2img reference、portable selection manifest、清理腳本與三張 contact sheet；runtime 只留模型/程式實際引用的 PNG，舊 bus 單圖與 `grid.png`/`crafting_grid.png` 已移除。尺寸/manifest/palette static test 不取代 fullscreen 實機視覺 gate。
