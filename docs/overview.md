# Overview

## What This Is

`magic_storage` — NeoForge 1.21.1 的一站式儲存 + 合成 mod。玩家用一個終端完成儲存、取用與合成：每種物品數量無上限，種類容量由有限 Storage Units 提供，或由 Creative Storage Unit 明確切換為無限；另有配方書式一鍵合成，以及由 Fuel 頁已安裝機器產生、合成時一次消費的「合成能量」。

概念源自 **Terraria 的 Magic Storage**(無限容量、配方書操作);NeoForge 的儲存網路 / grid UI / 物品處理等**技術模式參考 Refined Storage 2** 原始碼(見 `docs/notes.md`)。

## Key Concepts / Domain

- **Storage Core** — 網路中樞，所有計算（容量、合成、能量）都在 Core；Unit 只提供空間。永久資料從建立時起由 overworld `data/magic_storage_core_storages.dat` 的 `CoreStorageRepository` 擁有，Core 所在 chunk 只保存 `storageId` 與 `storageSchema`。庫存以每段最多 63 種的任意多段格式持久化，63 不是玩家容量上限。破壞、爆炸、創造模式或 Wrench 拆除含資料 Core 時，掉落物只攜帶一次性 recovery UUID 與摘要，放置時把同一 storage/network record 移到新 Core，不複製 payload；`/magic_storage recover_core` 只重發同一個未 claim capability。缺少、毀損、packed 或重複 attachment 一律 fail closed 並拒絕終端/自動化操作，不會製造空白 replacement。本次早期開發格式不遷移舊 inline Core NBT 或舊 recovery snapshot。
- **Storage Unit** — 六個有限 Tier 的儲存方塊獨立、不可逆、可並聯不限數量；T1→T6 以 copper cell、iron brace、gold/lapis lattice、diamond/quartz cross、prismarine/ender halo、netherite/amethyst crown 逐級增加 ornament complexity。`creative_storage_unit` 只由創造物品欄/指令取得、沒有配方/庫存/生成能力，en_us/zh_tw item tooltip 也明確標示無限種類且不生成物品；有效且目前已載入的單 Core 網路只要連上一個就以 explicit unlimited 種類容量運作，多個效果相同。移除或卸載只回到有限超額規則，不刪既有物品。
- **配方進程 / 指南** — T1 改以黑色混凝土粉末、銅、紅石與木桶製作，不再要求紫水晶；Core 與 T1–T6 依序引入黑曜石、玄武岩、鑽石/史萊姆、海磷晶/蜂蜜與獄髓。Patchouli 的雙語「配方總覽」集中顯示所有 Magic Storage 配方，每個配方只出現一次。
- **工具與終端進程** — Wrench 是 Brush + Tripwire Hook 的廉價建設工具。Storage Terminal 保持起步期但加入紅石火把控制件；Crafting Terminal 以金、銅、紅石和工作台成為早中期升級；Remote Terminal 再要求終界珍珠、黑曜石與指南針，作為較晚取得的移動能力。指南書本身註冊為 `magic_storage:guide_book` 並可合成。
- **Crafting Energy / Fuel categories** — 多個legacy能量池統一用 `Map<EnergyType, Long>`；第三方process stations使用descriptor-keyed work pool，無需擴充固定enum。NeoForge `magic_storage:machine_descriptor` 公開 registry 由server擁有順序與狀態；內建九個 Fuel-page descriptors分成三類：Furnace/Blast Furnace/Smoker/Campfire/Brewing Stand是可堆疊process machines，Crafting Table/Stonecutter/Smithing Table是最多各一個、可取回且只解鎖配方的instant stations，axe是不可安裝/取回的consumable input。Logical descriptor可同步1–64個concrete variants，每個variant有自己的normalized rational rate；PROCESS安裝總數可累積到`Integer.MAX_VALUE`且以one-count prototype + separate long count持久化，不受vanilla disk ItemStack codec限制。Core保存fractional remainder，換variant或live rate改變會重置不相容carry。Furnace同一格可接受vanilla與supported Iron Furnaces方塊，速度直接由目前cook-time config換算，不手寫tier表。Mekanism九種Factory family各接受basic machine及1/3/5/7/9-process factories，速率直接來自loaded tier contract；另外十二個deterministic single-block fluid/chemical descriptors使用各自exact machine/recipe work contract。Botania存在時另註冊Mana Pool、Runic Altar、Terrestrial Agglomeration Plate、Petal Apothecary與Elven Gateway五個max-one instant stations。Modern Industrialization存在時另註冊14個direct process descriptors，Bronze/Steel/Electric LV rate直接取loaded 2/4/8 EU tier contract。Ars Nouveau存在時另註冊Imbuement Chamber與Enchanting Apparatus兩個process descriptors，分別以100與210 station work執行。Powah存在時另註冊Starter–Nitro Energizing Rod variants，work/tick直接取loaded transfer config。Industrial Foregoing存在時另註冊Dissolution Chamber與Material Stonework Factory兩個process descriptors，rate與每份配方成本直接取loaded config/recipe contract。Create存在時另註冊Millstone、Crushing Wheel、Mechanical Saw、Spout與Item Drain五個process descriptors，以normalized 1 work/tick執行exact recipe work。Recipe preview先顯示actual installed variant，再依EMI/JEI相同的real-time 1000ms cadence輪播其他可用方塊；切換recipe或installed variant會重設到actual station，動畫不參與server執行。Fuel UI 先放獨立compact target bar，再以 Consumables / Timed Stations / Instant Stations 三個全寬category panels呈現；三者從標題下方填滿到Inventory label band，每個右側descriptor區依可用寬高形成multi-row pages，保留獨立滾輪並增加前後頁按鈕。 右下角搜尋切換啟用後，上半部改成跨三類與Fuel reserves的統一paged grid；搜尋重用名稱、`@mod`、`#tag`parser並比對polymorphic station的所有concrete variants，結果只影響client版面，不複製或改寫server儲存狀態。背包機器shift-click直接依exact descriptor路由，不受目前子分頁影響；near-cap安裝只接受剩餘容量並把未接受機器完整留在來源stack，timed數值hover顯示實際機器與總work/tick。Core 使用固定 256 格 bank，依 stable descriptor ID 持久化 machine/station stacks、work/remainder與 finite/explicit-infinite consumable state；descriptor缺少、category/ingredient不相容、超過上限或addon item暫時不存在時，原始entry NBT會留在repository record並於後續load重試。有限值為 remaining durability × (vanilla Unbreaking level + 1)，Unbreakable 轉為永不遞減的無限值。`ItemStack#getBurnTime(null)` 是 Fuel 真值，concrete cooking time 同時決定 process/Fuel 消耗。所有 mutation 在 server，client 只接同步。
- **Crafting/Fuel GUI** — `TerminalProfile` 以 capability 組合 Storage reduced profile 與 Crafting full profile，`TerminalLayout.forProfile(...)` 是唯一 geometry entry point；同一 immutable geometry 驅動 rendering、widgets、slot reconstruction、hitboxes、tooltips、scroll與 EMI exclusions。Storage/Craftable 的共用資源選擇器由 server 同步，保留物品、流體、能量、氣體、其他的穩定順序，但只循環目前已有 provider 的群組；base 不顯示氣體/其他，Mekanism chemical provider 註冊後才顯示氣體，Botania Mana、Ars Nouveau Source或獨立 addon kind 註冊後才顯示其他。`mekanism:chemical` persisted ID 與 registry canonical `magic_storage:chemical` 都歸入氣體。預設與中鍵重設都是物品，Fuel隱藏並拒絕切換。Storage中的non-item代表物不可取出；Craftable中的non-item代表物可選取exact typed-output recipe，但仍不會成為EMI item source。依 RS2 grid preference 慣例，Auto Focus及sort/order/search synchronization/resource view 與 Crafting 專用的 page/use-player-inventory/output/Fuel Target 由 physical-client `magic_storage-client.toml` 全域保存，所有實體與遠端終端共用；每次開啟仍由client封包送給server menu驗證與執行。搜尋欄原文直接由server解析一般名稱、`@mod`與`#tag`，Auto Focus獨立控制開啟終端時是否鎖定搜尋欄；Search Sync只控制Off、terminal→EMI或EMI雙向，不再代加前綴。搜尋文字、scroll offset、所選物品/配方與自適應列數不持久化，client也不保存任何Core資源狀態。搜尋輸入用2 tick trailing debounce；server一次compile query並重用Core normalized metadata cache，Craftable先過濾output才計算preview。Storage/Craftable 共用 server-owned exact amount、Name/Quantity/Mod/ID comparator與固定 slot-local scale；recipe workspace只消費 server-synced exact `RecipePresentation`，EMI存在時只在 diagram使用公開 widgets，ledger/footer始終由Magic Storage擁有。Cycle control統一 left-next/right-previous/wheel-down-next/wheel-up-previous/middle-reset，tooltip只列名稱與目前值；只有boolean on/off control畫彩色狀態燈，Craft Output/Fuel Target/sort/search/resource view等value selectors不亮燈。Fuel/recipe semantic workspace與player inventory共用vanilla light container grammar：raised panels、inset wells、slot frames、vanilla widgets與dark-gray text；world block palette不得套入GUI。Recipe diagram+ledger是footer上方的compact card；ledger最多四欄、由頂端對齊，目前可容納三列，超過八種required resources會進第三列；每格至少56px，available與required各用一行完整原字級文字且不截斷/縮字；`x1/x8/x64/Max`與navigation使用vanilla-style buttons。Fuel target仍有bounded foreground popup；三個Fuel descriptor grids各自使用adaptive columns × rows、page indicator與exact hover，並以每類64 descriptors回歸測試大量整合。每格為上方slot/icon、下方置中amount，空station顯示dim代表物。完整localized type-capacity訊息位於player inventory正右方的獨立info panel；無限狀態使用 Creative Storage Unit 圖示，有限狀態維持 T1 圖示，Instant Stations不再為它保留格位。第三方 descriptor 依 ResourceLocation 排序，menu opening payload 攜帶 server frozen snapshot，通用 consumable finite/∞ state 另由 clientbound packet 同步；詳見 [`machine-descriptor-api.md`](machine-descriptor-api.md)。
- **Terminal(三階)** — `storage_terminal`(T1 取存)、`crafting_terminal`(T2 配方+一鍵合成)、`remote_terminal`(T3 物品,同維度遠端存取；綁 exact Core UUID,同座標 replacement 不會被靜默接管；重綁不覆蓋其他 `CUSTOM_DATA`)。
- **Network / Bus automation** — Core 掃描/維護 Core ↔ Unit ↔ Terminal/Bus 連通;一般/指令/活塞放置會合併到 next-tick topology pass，安全範圍內增量成長，破壞/不確定拓樸時 full rebuild。Server 也會依 FULL chunk lifecycle延後、合併並bounded掃描loaded network boundary；Terminal/Bus只沿目前已載入且仍為network block的cached path工作，絕不為驗證連線載入chunk。Directional Import/Export每10 tick在正面各自嘗試一個item stack與一個bounded typed resource，並同時掃generic addon endpoint與native fluid/FE/optional chemical endpoint；成功item不會餓死typed，同一外部方塊的generic與native資源可同時被發現。Import在設定允許的side/null另暴露stable insert-only item/generic/native capabilities；Export只有Directionless才暴露filtered extract-only能力，item只是一格、最多正常stack的安全view，generic typed路徑明確拒絕Item kind。Mode、六面mask、unsided、automation、Allow/Deny與九格filter由server menu原子設定；owner/operator、距離、當前BE、stale revision與Wrench操作全部server驗證，legacy首次有效操作才claim owner。每個cached wrapper每次call都重驗BE identity、escrow、mode/side、filter、loaded Core path與conflict；缺Core時保持inert，設定面改變則invalidate capability。structured actor guard阻止same-network/inverse re-entry。Item與typed immutable plan都在commit前及external execute callback後重驗Bus/config/Core/network/topology/path/endpoint；Import不寫回detached source，Export會反向reclaim已接受量，detached target拒絕歸還的exact item差額也走recovery。rollback無法立即歸還時進Bus持久escrow，若Bus已被移除/取代則改產生owner-stripped recovery drop，絕不寫detached BE。能力暫停並先回流Core；一般/創造/Wrench/爆炸/direct-air/替換拆除都必須先回收或保存exact escrow，per-Bus recovery UUID與highest-priority drop protection避免later listener/event cancel吞掉drop或把兩次移除誤dedupe。

- **Typed-resource Core** — Item、fluid、NeoForge Energy、optional Mekanism chemical、optional Botania Mana、optional Ars Nouveau Source與addon kinds現在共用一個server-owned `StorageResourceLedger`；`ItemKey`只保留exact-item bridge，舊segmented item NBT是save/load相容邊界而非第二個live map。公開`magic_storage:resource_kind`、`resource_block_strategy`與`resource_container_strategy` registries，加上`StorageResourceKey`、`StorageResourceTransaction`與generic block capability，形成不授予任意Core callback的addon邊界。Storage/Crafting Terminal以代表物列出non-item exact key/long amount，並可在cursor上用一個supported fluid/FE/chemical/Mana/addon container原子存取；stacked container只處理一個，result必須能精確替換cursor或放入玩家背包，否則Core與玩家狀態整筆不變。icon不可被當物品抽出。Creative容量對全部kind無限；Bus主動/被動路徑支援generic與native typed endpoints。`deterministicResources` family可宣告1–9個exact consume/catalyst/tool inputs與多個primary/remainder outputs，所有Core item/non-item delta以同一transaction simulate→commit。Addon handler必須simulation無副作用、回傳`0..requested`且execute mutation精確等於回傳值；binary-incompatible optional bridge直接明確失敗，不silent fallback。Mekanism CI只驗一個代表版本，不建立多版本承諾；其他版本問題由使用者回報。External-machine send-and-wait仍不屬於本mod，multi-step planner仍是後續。完整契約見 [`typed-resource-storage.md`](typed-resource-storage.md) 與 [`recipe-family-api.md`](recipe-family-api.md)。
- **Assets / tools** — Runtime block/item icons維持vanilla 16x16與同一dark chassis/cyan-amethyst palette。T1–T6 ornament逐級華麗，Creative Storage Unit使用Replicate seed 71425候選與同family semantic cleanup完成cyan-amethyst infinity cell；全部以x=7.5水平鏡射並以2×2偶數中心校正。World casing預設使用ordinary 16x16 vanilla model；若client安裝可選Fusion 1.2.12，內建且自動啟用的connected-casing overlay才會以`pieced` 80×16 sheet相連，Creative Unit也納入全部12種network block predicates。item model仍是ordinary 16x16；Fusion只進Gradle隔離的client/data `fusionRuntime` source set，不進server/GameTest dependency也不bundle。Magic Storage Wrench加入`c:tools/wrench`並接受同tag的其他mod工具：右鍵旋轉directional bus，Shift+右鍵以正常loot path安全拆除network block；含資料Core透過同一repository record的recovery token移動。生成metadata/contact sheets只留在`art/texture-generation/`，static tests不取代fullscreen目視。
- **macOS GUI fullscreen validation** — Visual Prism sessions use Minecraft F11 through a macOS-only borderless Cocoa implementation, not GLFW monitor fullscreen: it never changes the desktop display mode. Each deterministic scenario builds only its current checklist's blocks, functions, targets, and navigation items. Repeatable setup is completed before handoff: visual-only crafting scenarios generate one matching server-owned repository record and Core reference with all required stored ingredients, installed stations, reserves, consumables, fluids, FE, and chemicals; the player inventory stays empty. Boot/guide has no network and bus retains only its interaction-specific fixture. The user opens pages/recipes and judges the result instead of installing machines or loading Fuel manually. The runner records the desktop mode before launch and rejects handoff if it differs at READY. User visual approval remains mandatory; normal close is F11 → visible titled window → Command-Q. See [`docs/macos-fullscreen-guide.md`](macos-fullscreen-guide.md) and [`docs/notes.md`](notes.md#prism-dev--manual-handoff).
