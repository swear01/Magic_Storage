## Module Boundaries

- 計算入口集中在 **Storage Core**；`StorageCoreBlockEntity`只持有repository attachment與runtime topology/cache，永久inventory/energy/machine/descriptor state由overworld `CoreStorageRepository`的`CoreStorageRecord`唯一擁有。Storage Unit只是空間,不自行計算。
- 網路狀態屬伺服器端;客戶端 GUI 一律靠封包同步,**不要**在客戶端保存儲存狀態。
- `ImportBusBlockEntity`與`ExportBusBlockEntity`的directional active front transfer在同一cooldown各自嘗試一個item stack與一個bounded typed resource，並同時發現generic addon與native fluid/FE/optional chemical endpoints；Import在設定side/null提供insert-only能力，Export只在directionless提供filter-gated extract-only能力。Generic typed capability拒絕Item kind，Export item view只有一格且最多正常stack。兩種Bus以schema 2 `BusConfiguration`持久化server-owned設定與typed exact filters；menu packet以container/position/revision/owner/operator/current-BE驗證後原子commit並invalidate capability。Stable cached wrappers保持物件身分但每次call重驗live BE、mode/side、automation、escrow與loaded Core path；no-Core時inert。Structured actor guard拒絕recursive same-network/inverse call；item與typed immutable plan都在commit及external execute callback後重驗Bus/config/Core/network/topology/path/endpoint；detached target拒絕反向reclaim的item差額與typed remainder都走exact recovery，detached Bus改產生owner-stripped recovery drop，並以per-Bus recovery UUID和highest-priority drop protection守住任何拆除。
- Runtime UI 語系檔 `en_us.json` / `zh_tw.json` 必須保持完全相同 key set；Patchouli localized content 仍依 Patchouli 自己的 locale tree 管理。
- 本 repo 是根工作區 `diy_minecraft_mods` 內的獨立 git repo,版控自成一套。
