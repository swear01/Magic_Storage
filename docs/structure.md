## Module Boundaries

- 計算入口集中在 **Storage Core**；`StorageCoreBlockEntity`只持有repository attachment與runtime topology/cache，永久inventory/energy/machine/descriptor state由overworld `CoreStorageRepository`的`CoreStorageRecord`唯一擁有。Storage Unit只是空間,不自行計算。
- 網路狀態屬伺服器端;客戶端 GUI 一律靠封包同步,**不要**在客戶端保存儲存狀態。
- `ImportBusBlockEntity` 的 active front pull 與 all-side passive insert-only capability 共用同一 server-side Core resolve/simulate/commit 契約；handler 不保存物品、不允許 extract，client/conflict/no-Core 全量退回，missing-Core BFS結果另共用10-tick negative cache。兩種Bus都以schema 1 `BusConfiguration`持久化server-owned設定，舊資料精確遷移、unknown/malformed raw資料保留並停用；`BusFilterPolicy`已接existing directional active/passive transfer，並讓automation disabled fail closed。Mode/side/unsided/owner設定與`ExportBusBlockEntity`無方向 capability尚未接線。
- Runtime UI 語系檔 `en_us.json` / `zh_tw.json` 必須保持完全相同 key set；Patchouli localized content 仍依 Patchouli 自己的 locale tree 管理。
- 本 repo 是根工作區 `diy_minecraft_mods` 內的獨立 git repo,版控自成一套。
